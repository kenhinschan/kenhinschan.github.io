Java的接口与抽象类

2019-09-11

接口与抽象类的区别

  • （1）抽象类可以有构造方法，接口中不能有构造方法。

  • （2）抽象类中可以有普通成员变量，接口中没有普通成员变量

  • （3）抽象类中可以包含静态方法，接口中不能包含静态方法

  • （4）一个类可以实现多个接口，但只能继承一个抽象类。

  • （5）接口可以被多重实现，抽象类只能被单一继承

  • （6）如果抽象类实现接口，则可以把接口中方法映射到抽象类中作为抽象方法而不必实现，而在抽象类的子类中实现接口中方法

接口和抽象类的相同点：

  • (1) 都可以被继承

  • (2) 都不能被实例化

  • (3) 都可以包含方法声明

  • (4) 派生类必须实现未实现的方法

为什么要使用接口

  • 平时项目当中经常使用到接口，那你有思考过为什么要使用接口？

  • 我们都知道接口里只做方法的声明，没有方法的具体实现

  • 接口的好处

    • 接口是产品经理等站在项目一层的人发起的动作，底层的码农必须按照接口要求进行实现

    • 业务功能扩展性

为什么要使用抽象类

  • 平时项目中抽象类使用的比较少，为什么要把类定义成一个抽象类？

  • 我们都知道抽象类里的可以有非抽象方法的具体实现，抽象方法的声明，抽象方法必须要被子类重写

  • 抽象类不能实例化

  • 多数开发者在项目中，都会用一般类代替抽象类

  • 抽象类的好处：唯一可以考虑到的就是，项目再重构时使用到（提醒你子类必须要重写抽象类的抽象方法）
